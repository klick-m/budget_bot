# Реализация (TDD Execution)

**ШАГ 1: ПОДГОТОВКА**
1. Прочитай `.context/CURRENT_TASK.md`.
2. Проверь ветку: `git branch`. (Если `main` — СТОП).

**ШАГ 2: ЦИКЛ TDD (Red -> Green)**
1. **RED:** Создай тест-файл в `tests/`, который воспроизводит требуемую логику. Запусти его (`pytest`), убедись, что он падает.
2. **GREEN:** Напиши реализацию в коде проекта.
3. **VERIFY:** Запусти тест снова.

**ШАГ 3: ЛОГИРОВАНИЕ**
* **ЕСЛИ ТЕСТЫ ПРОШЛИ:**
    1. Прочитай `.context/MEMORY_LOG.md`.
    2. Допиши в конец: `[Дата] ✅ UNIT TESTS PASSED: [Краткое описание]`.
    3. Сохрани файл.

**ШАГ 4: ПЕРЕДАЧА**
* Напиши: "Реализация завершена, тесты зеленые. Передаю в QA."
* **Команда:** `/workflow qa`
