# Планирование (Architectural Planning)

**ШАГ 1: ПОНИМАНИЕ ЗАДАЧИ**
1. Проанализируй запрос пользователя.
2. Прочитай текущую структуру проекта (используй `ls -R` или чтение файлов, если нужно).

**ШАГ 2: ОБНОВЛЕНИЕ ДОКУМЕНТАЦИИ**
1. Сформулируй техническое решение (какие файлы создать, какие изменить).
2. Запиши это решение в `.context/CURRENT_TASK.md`.
   - Используй формат Markdown Checklist (`- [ ]`).
   - Укажи конкретные имена файлов и методов.

**ШАГ 3: ПЕРЕДАЧА**
* Напиши: "План составлен и зафиксирован в CURRENT_TASK.md. Передаю слово Разработчику."
* **Команда:** `/workflow execute`
