# УСТАВ ПРОЕКТА (GLOBAL RULES)

## 1. БАЗОВЫЕ ЗАКОНЫ
1. **ЯЗЫК:** Все общение строго на РУССКОМ языке.
2. **ДИСЦИПЛИНА:**
   - Не меняй `brief.md` без разрешения Архитектора.
   - Никогда не удаляй код безвозвратно.
   - Никогда не коммить секреты (API ключи).

## 2. ИСТОЧНИКИ ДАННЫХ (DATA SOURCES)
1. **ЧТО ДЕЛАТЬ (ОПЕРАТИВНЫЙ ИСТОЧНИК):**
   - Файл `.kilocode/rules/memory-bank/current_task.md`.
   - Это твой "путевой лист". Если задачи нет в этом файле — ты её не делаешь.
2. **КАК ДЕЛАТЬ (КОНТЕКСТНЫЙ ИСТОЧНИК):**
   - Папка `.kilocode/rules/memory-bank/` (`architecture.md`, `tech.md`, `product.md`).
   - Ты ОБЯЗАН сверять свои решения с архитектурой и стеком проекта перед написанием кода.

## 3. ИЕРАРХИЯ
1. **User (Я)** — Заказчик.
2. **Архитектор** — Tech Lead. Ставит задачи в `current_task.md`.
3. **Кодер / Фронтенд** — Исполнители. Работают по TDD.
4. **QA / Ревьюер** — Контроль качества.

## 4. АЛГОРИТМ (DEFAULT LOOP)
1. **Понять** (Архитектор обновляет Memory Bank).
2. **Сделать** (Кодер пишет код).
3. **Проверить** (QA тестирует).
